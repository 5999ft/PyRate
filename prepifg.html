Prepifg Module¶

This Python module is part of the PyRate software package

This Python module converts interferogram input files to a common geotiff format with PyRate specific metadata headers. The module also implements multilooking/downsampling and cropping operations to reduce the size of the computational problem.

pyrate.prepifg.prepare_ifgs(raster_data_paths, crop_opt, xlooks, ylooks, thresh=0.5, user_exts=None, write_to_disc=True)[source]¶

Produces multilooked/resampled data files for PyRate analysis.

Parameters:
  • raster_data_paths – sequence of Ifg data paths (Currently DEMs are not supported)
  • crop_opt – integer cropping type option (see config)
  • xlooks – multilooking factor for the X axis
  • ylooks – Y axis multilooking factor
  • thresh (float) – (0.0, 1.0). Controls NaN handling when resampling to coarser grids. Value is the proportion above which the number of NaNs in an area is considered invalid. thresh=0 resamples to NaN if 1 or more contributing cells are NaNs. At 0.25, it resamples to NaN if 1/4 or more contributing cells are NaNs. At 1.0, areas are resampled to NaN only if all contributing cells are NaNs.
  • user_exts – CustomExts tuple with user sepcified lat long corners
  • verbose – Controls level of gdalwarp output
  • write_to_disc – bool, whether to write to disc during warp
pyrate.prepifg.resample(data, xscale, yscale, thresh)[source]¶

# TODO: make more efficient This is the slowest step in prepifg Resamples/averages ‘data’ to return an array from the averaging of blocks of several tiles in ‘data’. NB: Assumes incoherent cells are NaNs.

Parameters:
  • data – source array to resample to different size
  • xscale – number of cells to average along X axis
  • yscale – number of Y axis cells to average
  • thresh – minimum allowable proportion of NaN cells (range from 0.0-1.0), eg. 0.25 = 1/4 or more as NaNs results in a NaN value for the output cell.
pyrate.prepifg.warp(ifg, x_looks, y_looks, extents, resolution, thresh, crop_out, write_to_disc=True)[source]¶

Resamples ‘ifg’ and returns a new Ifg obj.

Parameters:
  • xlooks – int factor to scale X axis by, 5 is 5x smaller, 1 is no change.
  • ylooks – int as xlooks, but for Y axis
  • extents – tuple georeferenced extents for new file: (xfirst, yfirst, xlast, ylast)
  • resolution – [xres, yres] or None. Sets resolution output Ifg metadata. Use None if raster size is not being changed.
  • thresh – see thresh in prepare_ifgs().
  • verbose – True to print gdalwarp output to stdout
  • write_to_disc – bool, whether to write to disc during warp
